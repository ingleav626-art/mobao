# AI 托管功能计划

## 目标

在游戏状态栏添加"AI托管"按钮。**p2 是第四个AI**，平时静默积累上下文，唤醒后与其它AI完全一致。

---

## 核心设计：静默积累 + 唤醒执行

p2 始终作为第四个AI存在，只是决策来源不同：
- **未唤醒时**：玩家操作 → 写入 p2 的记忆（如同 AI 自己做了这个决策）
- **唤醒时**：AI 自主决策，走与 p1/p3/p4 完全相同的代码路径
- **结算时**：无论是否唤醒，游戏数据都写入 p2 的经验本

---

## 静默积累（始终运行）

### 每轮玩家操作时写入 p2 记忆

**玩家使用技能/道具时**：
```
记录到 p2 的 aiConversationByPlayer：
{ round, bid: null, skill: "技能-拓影侦测", item: "无", thought: "玩家手动操作", result: "" }
```

**玩家提交出价时**：
```
更新 p2 最近一条记录的 bid 字段
更新 result：上一轮结算结果
```

**涉及文件**：`main.ts`、`bidding/index.ts` — 在 `playerBid()`、技能/道具使用处添加写入

### 结算时写入 p2 经验本 + 滑动窗口

**`memory.ts` — `pushRunSettlementContextToAi`**：

当前只写 `!p.isHuman` 的玩家。改为：
```typescript
const targetPlayers = this.players.filter((p) => !p.isHuman || (p.id === "p2" && this.autoPlayEnabled))
```

但游戏统计数据（仓库价值、利润等）应该**无论是否托管都写入** p2 的经验本：

```typescript
// 始终写入 p2 的游戏数据
if (this.aiCrossGameMemory["p2"]) {
  // 更新 stats: totalGames, winRate, avgProfit, warehouseValue 等
}
```

**p2 跨局消息滑动窗口**：

```typescript
// 每次结算时检查
const p2Messages = this.aiCrossGameMessagesByPlayer?.["p2"]
if (Array.isArray(p2Messages) && p2Messages.length >= contextLength) {
  if (this.autoPlayEnabled) {
    // 可以调用LLM → 触发总结（在reflection中处理）
  } else {
    // 无法总结 → 丢弃最旧的一局
    p2Messages.shift()
  }
}

### 反思/总结时包含 p2

**`reflection.ts` — `triggerAiReflection`**：

`aiPlayers` 过滤：
```typescript
const aiPlayers = this.players.filter((p) => 
  !p.isHuman || (p.id === "p2" && this.autoPlayEnabled)
)
```

- 唤醒状态：p2 参与反思，生成经验教训
- 未唤醒状态：p2 不参与反思（因为没有 LLM 上下文），但游戏数据已在结算时写入

---

## 唤醒执行（auto-play 时）

### 触发时机

**`processAiDecisions`**：如果 `autoPlayEnabled`，把 p2 加入 `aiPlayers` 列表，与 p1/p3/p4 完全并行。

### 决策流程

p2 走与其它 AI 完全相同的路径：
1. `requestAiLlmPlan(p2)` 或规则 AI
2. `processSingleAiIntelAction(p2, plan, llmPlan, ...)`
3. `executeAiIntelAction("p2", plan)`
4. `setPlayerBidReady("p2", true)`
5. `playerBid()` 提交出价

### 上下文来源

p2 的 LLM 上下文自然包含：
- `getAiConversationMessages("p2")` — 静默积累的跨局消息
- `getAiCrossGameMemory("p2")` — 静默积累的经验本
- `aiCrossGameMessagesByPlayer["p2"]` — 跨局历史
- `pendingNextRunAiSummaryByPlayer["p2"]` — 总结

## p2 跨局记忆管理

### 问题

p2 的托管可能不是每局都开。跨局记忆有上限（contextLength），达到上限需要总结才能继续积累。但总结需要 LLM（托管中才有），如果连续几局不开托管，记忆就会超限。

### 解决：滑动窗口 + 条件总结

**每次结算时检查 p2 的跨局记忆**：

```
如果 p2 的跨局消息数 >= contextLength:
  如果 autoPlayEnabled（可以调用LLM）:
    触发 p2 的总结 → 生成总结 → 清空跨局消息
  否则（无法总结）:
    滑动窗口：丢弃最旧的一局消息
    aiCrossGameMessagesByPlayer["p2"].shift()
```

这样：
- 开托管时：正常总结，记忆精炼
- 不开托管时：滑动窗口丢旧的，保证不超限
- 下次开托管时：从当前窗口继续，不会丢失最近的记忆

**涉及文件**：`memory.ts` — `pushRunSettlementContextToAi` 中添加 p2 的滑动窗口逻辑

## 边界情况

| 场景 | 处理 |
|------|------|
| 第3回合才开启托管 | p2 已有前2回合的静默记录，AI决策时有上下文 |
| 托管中取消 | 当前决策继续执行，下回合恢复手动，后续操作继续静默写入 |
| 从未开启托管 | p2 的记忆持续积累，只是不被使用 |
| 已用过道具/技能 | `executeAiIntelAction` 自然跳过工具，只出价 |
| 超时未返回结果 | 出价为 0，与普通 AI 一致 |
| 结算时未托管 | 游戏数据仍写入 p2 经验本 |
| 结算时已托管 | 游戏数据 + 反思都写入 |
| 跨局记忆满+未托管 | 滑动窗口丢弃最旧一局 |
| 跨局记忆满+已托管 | 触发总结，清空跨局消息 |

---

## 验证方法

1. 玩2局手动操作 → 第3局开启托管 → 检查 p2 是否有前2局的上下文
2. 托管中玩2局 → 检查 p2 的经验本、反思是否正确
3. 取消托管 → 继续手动操作 → 检查 p2 记忆是否继续积累
4. AI决策面板 → 确认 p2 的决策正确显示
5. 设置中查看 p2 记忆 → 确认数据正确
6. 连续5局不开托管 → 检查 p2 跨局消息是否滑动窗口正确丢弃
7. 第6局开托管 → 检查是否触发总结
