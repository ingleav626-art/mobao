## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 3D 翻转性能问题 | 300 格同时翻转卡顿 | 分批翻转 + 仅翻转视口内 + `will-change` 优化 |
| 联机回归测试环境 | 需要多设备/多浏览器 | 优先在 Android WebView + Chrome 桌面测试 |
| 记忆分离迁移 | 旧记忆格式不兼容 | 自动迁移 + 保留旧键作为备份 |
| Mixin 依赖梳理 | 发现循环依赖 | 逐步解耦，不一次性重构 |
| 音效文件获取 | 版权/质量/风格问题 | 优先使用免费音效库（Freesound/Mixkit） |
| 明信片功能范围蔓延 | 可能过度设计 | 首版仅做背景切换，不做复杂交互 |
| 提示词优化过度 | 可能影响 AI 决策质量 | A/B 测试对比优化前后效果 |
| 多模型测试复杂度 | 测试工作量大 | 先测试核心模型，逐步扩展 |
| 语音系统规划过早 | 可能与后续实现脱节 | 仅做架构预留，不实现具体功能 |

## v2.0 前置条件

完成本计划后，项目应满足以下 v2.0 前置条件：

1. **功能完整**：角色系统、联机系统、音效系统、动画系统全部验收通过

5. **无高优 Bug**：已知高优先级 bug 全部修复
6. **设置有效**：所有设置项功能验证通过
7. **语音预留**：语音系统架构规划完成

v2.0 的核心是**模式系统**（线索竞拍/2v2/斗蛐蛐），依赖以上前置条件才能稳定开发。
