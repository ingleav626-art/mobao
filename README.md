# 摸宝仓库 — 古董竞拍·暗仓探秘

一款基于 Phaser 3 的仓库摸宝竞拍游戏。你扮演寻宝者，在堆满未知藏品的仓库中，用有限的信息估算价值，与 LLM 或真人对手竞价博弈。

---

## 快速开始

**手机端**：下载 APK 安装即可。

**电脑端**：

```bash
npm install
npm run dev
```

浏览器打开 `http://localhost:5173` 就能玩，不需要手机。但提醒：我暂时没有做电脑端的css适配，所以游戏界面可能会有问题。

## 缘起

在网上看到类似的仓库摸宝游戏时十分想玩，但发现要钱。于是我想，不如自己做一个，还免费。仅用 3 天不到就完成了基本玩法，但和规则 AI 玩实在太无趣了，所以加入了 LLM —— 这才是这个项目的真正起点。我想要探索 AI 和游戏到底能结合到什么程度。

## 游戏玩法

说简单也简单：每局随机仓库，每轮你可以探查藏品信息，然后跟对手竞价。最后出价最高者拍下仓库，根据仓库真实总价值结算盈利或亏损。

难的是：信息永远不完整，对手会骗（当然目前ai还是太诚实了），你可能花大价钱买回一堆破烂，也可能低价捡漏一夜暴富。

具体步骤不废话了，玩上两把就懂。

## LLM AI —— 这个项目的核心

现在的 AI 已经能自己利用游戏中的信息出价、选道具，根据场上情况调整策略，但是心思还是太单纯了。玩家可以自备 API Key，在设置里调整 AI 的记忆和行为：

- 让 AI 在每局结束后反思总结，越玩越聪明
- 调整上下文长度，控制 AI 的"记忆"范围
- 查看 AI 的决策过程和思考逻辑，也可以翻它的"反思本"
- 导出 AI 的反思记录，跟朋友分享

后续计划加更多 AI 功能：TTS 语音、局内聊天、表情包、自定义 AI 人格……让每个 AI 都像个活人。

## 当前状态

坦诚地说，游戏还是个半成品，但勉强可玩：

- **角色**：当前 3 个，只有两个有动态立绘（甚至只有一个是能看的）。我做好了 12 个角色的立绘和功能设计，但动态立绘用 AI 生成成本较高，暂时搁置
- **道具**：没有图片（后续补，这个不难），价格和稀有度是临时的（测试时全 0 元）
- **技能和道具设计**：临时性的，极度不平衡，为了测试都没限制使用次数。后续会让每个角色有自己的玩法和风格
- **联机模式**：可用但不稳定。重构后很多功能没完整测试，可能出现 bug
- **新手教程**：还没做，考虑后续加入

## 计划路线图

| 模块 | 计划 |
|------|------|
| AI | TTS 语音、局内聊天、表情包、自定义人格 |
| 角色 | 逐步扩充至 12 个，每个有独立立绘和玩法 |

该项目使用的ai工具很多：包括但不限于：claude code，trae，opencode等
该项目使用的ai模型也很多：主要为glm系列模型，以及deepseek系列模型。但是项目最开始的3天基础可玩版本是通过chatgpt-5.3codex模型搭起来的。后续因为没法白嫖了就换国产模型继续白嫖。可以说，现在你看到的所有功能我都没有花钱。所以项目所需要的时间自然会较长。
