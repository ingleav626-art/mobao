# 摸宝仓库 — 古董竞拍·暗仓探秘

一款基于 Phaser 3 的仓库摸宝竞拍游戏。玩家选择角色，在随机生成的仓库中探查古董藏品与珠宝首饰，与 AI 或真人玩家博弈出价，在有限轮次内以低于仓库真实总价值的成交价盈利。

## 游戏机制

### 核心玩法

- **角色选择**：开局前从 3 个角色中选择，每个角色拥有独立的主动技能与被动能力，直接影响探查策略与收益
- **仓库探查**：每局随机生成仓库，藏品以网格形式分布（12×25 格），品质从粗品到绝品共五档，大小从 1×1 到 4×1 不等
- **多轮竞拍**：玩家与 3 个 AI 或真人竞拍者轮流出价，非最终轮可触发"直接拿下"（出价达到第二名 × directWinRatio）
- **技能与道具**：每轮可使用角色专属技能和携带的道具揭示藏品信息（轮廓、品质、全信息）
- **公共事件**：每局随机出现市场传闻、拍卖行消息等公共事件，影响决策判断

### 竞拍流程

```
大厅选角 → 选择地图 → 进入仓库 → 轮间阶段（出价+技能/道具）→ 出价揭示 → 下一轮/结算
```

1. **轮间阶段**：玩家可使用技能/道具探查藏品，然后提交出价
2. **出价揭示**：所有玩家同时提交出价后公开，判断是否提前结束
3. **提前结束条件**（非最后一轮）：第一名出价 > 第二名出价 × (1 + directWinRatio)，则第一名直接赢得仓库
4. **正常结束**：最终轮出价最高者赢得仓库
5. **结算揭示**：逐一揭示仓库中所有藏品的真实信息，计算盈亏

### 分红与门票

- **分红机制**：拍下者亏损时，非拍下者各获得亏损额 15% 的分红（鼓励欺诈对手高价拍下）
- **门票机制**：拍下者盈利时，非拍下者各被扣除盈利额 5% 的门票（鼓励积极竞拍）

### 地图配置

| 地图 | 图标 | 描述 | 轮数 | 直接拿下倍率 | 特点 |
|------|------|------|------|-------------|------|
| 废弃仓库 | 🏚️ | 均衡配置，适合入门 | 5 | 0.20 | 品质均衡，品类均衡 |
| 珍宝密室 | 💎 | 高品质藏品更多，竞争更激烈 | 6 | 0.25 | 珍品/绝品权重高，玉器/书画多 |
| 废品角落 | 🗑️ | 低品质居多，考验捡漏眼光 | 4 | 0.15 | 粗品/良品权重高，木器多 |
| 书斋雅集 | 📜 | 书画金石为主，文人之选 | 5 | 0.20 | 书画/金石权重高，精品多 |

## 角色系统

角色系统是 v1.5 新增的核心模块，每个角色拥有独立的主动技能和被动能力，直接影响游戏策略。

### 角色列表

| 角色 | 描述 | 主动技能 | 被动能力 |
|------|------|---------|---------|
| **鉴定师** | 精准识宝，稳扎稳打 | 玉脉鉴质 — 优先对玉器揭示2件品质格 | 盈利加成+10% |
| **探子** | 眼观六路，广撒大网 | 拓影侦测 — 揭示3件藏品的完整轮廓 | 轮廓揭示+1 |
| **觅踪者** | 洞察秋毫，直取要害 | 鉴踪直取 — 直接揭示轮廓最大的1件藏品的所有信息 | 轮廓探测优先轮廓最小 |

### 角色选择流程

1. 大厅点击"单人模式" → 选择地图 → 进入角色选择页面
2. 角色选择页面展示角色卡片列表，点击选中
3. 选中后右侧展示角色详情：主动技能描述 + 被动能力描述
4. 确认选择后进入游戏

### Live2D 预览

角色选择页面支持 Live2D 风格的角色预览动画，采用双视频无缝循环方案：

- 使用两个 `<video>` 元素（A/B）交替播放
- 当前视频接近结尾时预热备用视频，实现无缝切换
- 桌面端预热时间 2 秒，切换间隔 33ms；移动端预热时间 5 秒，切换间隔 4 秒
- 支持 `requestVideoFrameCallback` 优化，不支持时自动降级为轮询
- 加载失败自动重试（最多 3 次），超时 5 秒触发重试

### 角色数据持久化

- 选择的角色保存至 `localStorage`（`mobao_selected_character_v1`）
- 下次进入游戏自动恢复上次选择
- 角色被动能力在结算时生效（如鉴定师的盈利加成）

### 相关文件

| 文件 | 功能 |
|------|------|
| `scripts/game/data/characters.js` | 角色数据定义（3 个角色：鉴定师、探子、觅踪者） |
| `scripts/game/data/character-system.js` | 角色系统逻辑（选择、被动效果计算、持久化） |
| `scripts/game/lobby/character-select.js` | 角色选择 UI（卡片列表、Live2D 预览、确认选择） |

## AI 系统

### 双引擎架构

游戏 AI 采用规则引擎 + LLM 大模型的双引擎架构，可随时切换：

- **规则 AI**（`ai/bidding.js`）：基于人格参数的规则化出价引擎，每个 AI 有独立的激进/纪律/跟风/虚张声势等参数
- **LLM AI**（`ai/decision.js` + `llm/`）：支持多家大模型，让 AI 基于上下文做出决策

### 规则 AI 人格参数

每个 AI 玩家拥有独立的人格参数，控制出价行为：

| AI | 人格 | 激进 | 纪律 | 跟风 | 虚张声势 | 失误率 | 锚点范围 |
|----|------|------|------|------|---------|--------|---------|
| 左上AI | 稳算师 | 0.58 | 0.86 | 0.32 | 0.18 | 0.04 | 1.24~1.72 |
| 右上AI | 猛冲客 | 0.84 | 0.62 | 0.56 | 0.30 | 0.08 | 1.42~1.98 |
| 右下AI | 机变派 | 0.54 | 0.88 | 0.26 | 0.24 | 0.05 | 1.20~1.78 |

参数说明：

- **激进程度**：影响出价积极性和冒险倾向
- **纪律性**：影响出价稳定性和理性程度
- **跟风倾向**：对市场参考价的敏感度和模仿程度
- **虚张声势**：前中期故意藏价的概率
- **失误率**：做出非理性出价的概率
- **锚点范围**：初始锚点出价相对于市场参考价的倍数范围
- **开局抬价率**：开局抬价时的加成比例
- **群体偏差**：群体多样性调整中的倾向
- **预期弹性**：调整心理预期出价时的适应速度
- **退却因素**：感到过热时的回撤程度
- **无信息调整**：线索率较低时出价的随机调整范围

### LLM AI 决策流程

```
1. 构建上下文 → 2. 调用 LLM API → 3. 解析 JSON 响应 → 4. 校验合法性 → 5. 执行/回退
```

- **两段式流程**：initial 阶段可出价+选技能/道具；若执行了工具则进入 follow-up-after-tool 阶段
- **纠错机制**：LLM 输出不合法时最多重试 2 次，仍失败则回退到规则 AI 结果
- **钱包归一化**：LLM 出价会被归一化到步长和钱包上限范围内

### LLM 提供商

| 提供商 | 默认模型 | 默认端点 |
|--------|---------|---------|
| DeepSeek | deepseek-v4-flash | /api/deepseek/chat/completions |
| OpenAI | gpt-4o-mini | https://api.openai.com/v1/chat/completions |
| 通义千问 | qwen-turbo | https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions |
| 智谱GLM | glm-4-flash | https://open.bigmodel.cn/api/paas/v4/chat/completions |
| Kimi | moonshot-v1-8k | https://api.moonshot.cn/v1/chat/completions |

支持自定义提供商：可添加任意兼容 OpenAI API 格式的 LLM 服务。

### AI 模块拆分

| 模块 | 功能 |
|------|------|
| `bidding.js` | 规则化出价引擎，人格参数控制 |
| `decision.js` | LLM 决策调度，技能/道具选择，决策面板快照 |
| `intel.js` | 情报分析，AI 私有信息池管理，技能/道具执行上下文 |
| `memory.js` | 跨局记忆存储与恢复（localStorage 持久化） |
| `reflection.js` | 局后反思生成（调用 LLM 生成 200 字内反思总结） |
| `wallet.js` | AI 钱包管理，出价归一化校验 |

### 跨局记忆

- 开启后 AI 会记住历史每局的结果与反思
- 每个玩家保留最近 30 条本局对话历史和 20 条跨局记忆
- 局后反思功能：结算后调用 LLM 生成 200 字内的反思总结，存入跨局记忆
- 跨局记忆在 LLM 决策时注入到 prompt 中，帮助 AI 积累经验

## 联机系统

### 架构

- **服务端**：Node.js WebSocket 服务器（`lan/server/server.js`）
- **客户端**：浏览器端桥接层（`lan/client/lan-bridge.js`）
- **协议**：JSON 消息通信（`lan/shared/protocol.js`），协议版本 1

### 通信协议

**客户端 → 服务端**

| 消息类型 | 说明 |
|---------|------|
| `lan:ready` | 玩家就绪 |
| `lan:bid:submit` | 提交出价 |
| `lan:action:use` | 使用技能/道具 |
| `lan:settle:request` | 请求结算 |
| `lan:pause:toggle` | 暂停/恢复 |
| `lan:ping` | 心跳 |

**服务端 → 客户端**

| 消息类型 | 说明 |
|---------|------|
| `lan:game:init` | 游戏初始化 |
| `lan:round:start` | 回合开始 |
| `lan:round:bid-ack` | 出价确认 |
| `lan:round:bid-reveal` | 出价揭示 |
| `lan:round:result` | 回合结果 |
| `lan:game:settle` | 游戏结算 |
| `lan:game:settle-reveal` | 结算揭示 |
| `lan:game:settle-final` | 最终结算 |
| `lan:pause:state` | 暂停状态 |
| `lan:player:left` | 玩家离开 |
| `lan:error` | 错误消息 |
| `lan:pong` | 心跳回复 |

### 功能

- 创建/加入房间（支持公开/私密房间、密码保护）
- 实时同步出价、技能使用
- 主机端 AI 补位（人数不足时自动填充 AI 玩家）
- 结算同步与再来一局
- 局域网服务器自动发现

### 启动联机服务器

```bash
cd lan/server
npm install
node server.js
```

## 技术架构

### 技术栈

| 技术 | 用途 |
|------|------|
| Phaser 3.90 | 游戏引擎（仓库渲染、动画、交互） |
| 原生 JavaScript | 无框架，IIFE + Mixin 模式 |
| localStorage | 所有数据持久化（设置/战绩/记忆/角色等） |
| LLM API | 可选的 AI 大模型决策（5 家内置提供商） |
| WebSocket | 联机通信 |
| Web Audio API | 音频播放 |
| Android WebView | 可打包为 Android APK |

### 架构模式

- **IIFE 封装**：每个模块使用 IIFE 封装，挂载到 `window` 全局对象
- **Mixin 组装**：`WarehouseScene` 通过 `Object.assign` 合并 16 个 Mixin 到原型
- **Bridge 模式**：`shop.js`、`battle-record.js`、`settlement.js` 作为桥接模块，通过 `.call(this)` 借用场景上下文
- **全局命名空间**：`window.MobaoXxx` 组织模块（MobaoConstants、MobaoUtils、MobaoSettings、MobaoWarehouse、MobaoAi、MobaoUi、MobaoLobby、MobaoLan 等）

### 数据存储键

| 键名 | 用途 |
|------|------|
| `mobao_settings_v2` | 游戏设置 |
| `mobao_player_money_v1` | 玩家资金 |
| `mobao_ai_llm_switch_v1` | LLM 开关与配置 |
| `mobao_battle_records_v1` | 战绩记录（最近 20 局） |
| `mobao_ai_memory_v1` | AI 跨局记忆 |
| `mobao_app_state_v1` | 应用全局状态 |
| `mobao_selected_character_v1` | 选中的角色 |
| `mobao_shop_inventory_v1` | 商店库存 |
| `mobao_shop_refresh_date_v1` | 商店刷新日期 |
| `mobao_audio_settings` | 音频设置 |
| `mobao_lan_name` | 联机玩家昵称 |

## 运行方式

### 本地运行

直接用浏览器打开 `index.html` 即可运行（需 HTTP 服务器）：

```bash
# 使用 Python 简易服务器
python -m http.server 8080

# 或使用 Node.js
npx serve .
```

### LLM API 代理

如果需要使用 LLM 功能但遇到跨域问题，可启动本地 CORS 代理：

```bash
node proxy-server.js
# 代理运行在 http://localhost:3000
# 前端请求发到代理，通过 x-target-url header 指向真实 API
```
