========================================================================
第一层：静态（永不变化，永不清理，始终在开头）
========================================================================

[1. 系统提示词]
   role: system
   身份与目标、游戏机制、字段参考、硬约束、策略建议、输出格式

[2. 图鉴摘要]
   role: user
   【图鉴摘要】totalArtifacts、qualityPriceGuide（首轮）/ qualityPriceRangeTable（后续轮）

========================================================================
第二层：持久记忆（不随上下文清空删除，跨局保留）
========================================================================

[3. 经验本] ──── 可选
   role: user
   【经验本】
   lessons / strategies / praises（JSON格式）
   条件：有经验本数据时插入
   清理：仅手动删除
   说明：首次反思生成后插入，之后永久保留。
         反思关闭仅停止更新，不删除已插入的内容。

[4. 上期总结] ──── 可选
   role: user
   【上期总结】
   最近 context_length 局的胜率、关键教训、出价规律等摘要
   条件：多局开启且有上期总结数据时插入
   清理：下次上下文清空时被新总结覆盖

========================================================================
第三层：动态（按 context_length 滑动清理）
========================================================================

[5. 上局结算] ──── 可选
   role: user
   【上局结算】
   上局的出价结果、行动记录
   条件：仅多局关闭（context_length=1）时，放在动态部分开头
   清理：随动态部分清理
   说明：多局开启时不需要，因为 AI 能从累积的对局历史里看到。

[6. 对局信息]
   role: user
   【角色与工具】character + skills + items
   【其他玩家公开信息】对手角色、公开行动
   【游戏状态】round / wallet / Previousbid / currentLeader
   【私人情报】aggregate + highValueTracks
   【行动约束】canBid / availableSkills / availableItems
   【出价历史】仅首轮有
   【公共事件】仅首轮有
   【轮次公开状态】仅后续轮有，替代出价历史+公共事件
   【可用工具】仅后续轮有
   【轮次信息】仅后续轮有

[7. AI 回复]
   role: assistant
   {"bid": N, "skill": "...", "item": "...", "thought": "..."}

[8. 工具结果] ──── 可选
   role: user
   【工具调用上下文】toolResultSummary + initialDecision
   条件：执行了技能或道具后追加

[9. AI 调整出价] ──── 可选
   role: assistant
   根据工具结果修正后的出价决策
   条件：工具执行后

[10.轮次结算]
   role:user
   [上一轮结算]增量消息【轮次公开状态】仅后续轮有，替代出价历史+公共事件

中间的格式循环

[11. 对局结算] ──── 可选
   role: user
   【本局结算】
   本局的仓库信息和最终游戏结果
   条件：多局开启且未达 context_length 上限时追加在本局末尾
   说明：达上限时不追加，结算数据丢弃。
         多局关闭时也不追加（因为下局开头已有上局结算）。

========================================================================
清空触发条件
========================================================================

多局开启时（context_length >= 2）：
  当动态部分的局数达到 context_length 时清空。
  清空时刻：
    - 生成上期总结（覆盖旧的）
    - 更新经验本（追加新条目）
    - 上局结算丢弃
  清空后下一局：
    - 动态部分重新从对局信息开始累积
    - 经验本和上期总结继续保留

多局关闭时（context_length = 1）：
  每局结束时清空动态部分。
  清空时刻：
    - 更新经验本（每局追加）
    - 上期总结不生成
  清空后下一局：
    - 上局结算放在动态部分开头
    - 经验本保留

========================================================================
经验本更新规则
========================================================================

- 多局开启：每 context_length 局清空时更新一次（跟随总结时机）
- 多局关闭：每局更新
- 反思关闭：仅停止更新，已插入的经验本不删除
- 经验本清除：仅玩家手动操作（清空AI记忆）时删除
